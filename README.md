### Задание 1 
Крупная строительная компания, которая также занимается проектированием и девелопментом, решила создать правильную архитектуру для работы с данными. Ниже представлены задачи, которые необходимо решить для каждой предметной области.

Какие типы СУБД, на ваш взгляд, лучше всего подойдут для решения этих задач и почему?


1.1. `При планировании бюджета проектов, создании аналитических финансовых отчётов и оценке возможных рисков необходимо применять надёжную систему управления базами данных. Такая система должна обеспечивать структурированное хранение информации и её сохранность.
Среди различных типов СУБД особое преимущество имеют реляционные системы. Их главное достоинство заключается в поддержке ACID-свойств, включающих: атомарность операций, согласованность данных, изолированность транзакций, долговечность сохранённой информации.
Кроме того, реляционные СУБД предоставляют широкие возможности для работы со сложными SQL-запросами и позволяют устанавливать различные ограничения для поддержания целостности данных. Это делает их идеальным решением для задач финансового анализа и управления проектами.`

1.2. `Для Лендинга и CRM при условиях что СУБД должна  быть гибкой и быстрой, идеально подходят  NoSQL-СУБД.  
Решение на NoSQL-СУБД обеспецивает: 
-	гибкость схемы данных (например, для хранения неструктурированных или быстро меняющихся данных);
-	масштабируемость (горизонтальное масштабирование для обработки больших объёмов данных и высокой нагрузки);
-	скорость записи данных.`
  
1.3. `Для базы по корпоративным нормам и строго структурированной информации идеально подходят реляционные СУБД они обеспечивают:
-	понятную схему данных;
-	простоту создания отчетов;
-	надежную защиту информации;
-	легкость обучения персонала.`

1.4. `Для формирования маршрутов доставки курьеров лучше всего подходят графовые СУБД.
Графовые СУБД особенно эффективны в случаях, когда требуется анализ сложных взаимосвязей между данными и быстрый поиск связанных объектов. Они становятся все более востребованными в современных проектах благодаря своей гибкости и производительности при работе со связанными данными.`

### Задание 2 Транзакции

 `Этапы транзакции:
1. Начало транзакции
- система получает запрос на пополнение
- открывается сессия работы с базой данных
2. Инициирование операции
- пользователь выбирает способ пополнения (онлайн-банк, терминал, сайт оператора)
- ввод необходимых данных (номер телефона, сумма)
3. Валидация данных
- проверка корректности номера телефона
- проверка допустимости суммы (минимум/максимум)
- проверка достаточности средств на источнике платежа

4. Авторизация платежа
- подтверждение операции пользователем (код из SMS, подпись в чеке)
- проверка статуса карты/счета
- проверка лимитов на операции
5.	Обработка платежа
- блокировка суммы на исходном счете
- проверка возможности зачисления на целевой счет
- обновление балансов
6.	Фиксация результата
- зачисление средств на счет телефона (  если все операции прошли успешно)
- формирование подтверждающих документов
- уведомление пользователя об успешном завершении

7.	Завершение транзакции
- закрывается сессия работы с базой данных`
